<div align="center"> <img src="https://cdn.jsdelivr.net/gh/devicons/devicon/icons/python/python-original.svg" width="90" /> <h1>OCR System</h1> <p><strong>Configurable Python OCR platform for extracting document numbers and barcodes from PDF documents.</strong></p> <p> Modular architecture supporting multiple document types using engine-based processing and JSON configuration. System behavior can be changed without modifying source code. </p> <p> <img src="https://img.shields.io/badge/Language-Python-blue" /> <img src="https://img.shields.io/badge/Architecture-Modular-success" /> <img src="https://img.shields.io/badge/Input-PDF-lightgrey" /> <img src="https://img.shields.io/badge/Focus-Automation-black" /> </p> </div> <hr/> <h2>Overview</h2> <p> OCR System is a configurable automation platform designed to extract numeric identifiers and barcodes from PDF documents. It supports multiple document formats and processing strategies using modular engines and JSON-based configuration. </p> <p> The system is designed for production use where reliability, configurability, and predictable behavior are required. New document types can be added without changing the core engine logic. </p> <hr/> <h2>Supported Engines</h2> <ul> <li> <strong>Delivery Note</strong><br/> Landscape documents. Extracts numeric identifiers from a fixed region using OCR. </li> <li> <strong>Purchase Order</strong><br/> Portrait documents. Extracts numeric identifiers from a fixed region using OCR. </li> <li> <strong>Shipment</strong><br/> Multi-page documents. Extracts the identifier from the relevant page only. </li> <li> <strong>Picking Slip</strong><br/> Barcode-based extraction using Code128 and similar formats. </li> </ul> <hr/> <h2>Core Features</h2> <ul> <li>Folder monitoring using watchdog</li> <li>PDF to image conversion using pdf2image</li> <li>OCR using EasyOCR and Tesseract</li> <li>Barcode decoding using pyzbar</li> <li>ROI based extraction</li> <li>Automatic file renaming</li> <li>Error handling and debug image capture</li> <li>Fully configurable via JSON</li> </ul> <hr/> <h2>Project Structure</h2> <ul> <li><strong>core</strong>, shared utilities and helpers</li> <li><strong>engines</strong>, document specific processors</li> <li><strong>configs</strong>, example configuration files</li> <li><strong>examples</strong>, sample input files</li> <li><strong>docs</strong>, architecture diagrams and documentation</li> </ul> <hr/> <h2>How It Works</h2> <ol> <li>You configure paths and OCR settings in a JSON file</li> <li>The engine watches the input folder</li> <li>When a PDF appears, the system waits until the file is stable</li> <li>The PDF is converted into an image</li> <li>The ROI region is extracted</li> <li>OCR or barcode decoding runs</li> <li>The extracted number becomes the new filename</li> <li>The renamed PDF moves to the output folder</li> <li>Failed files move to the error folder with debug images</li> </ol> <hr/> <h2>Requirements</h2> <ul> <li>Python 3.9 or newer</li> <li>Poppler installed for pdf2image</li> <li>Tesseract installed if using OCR engines</li> </ul> <h3>Python Packages</h3> <ul> <li>watchdog</li> <li>pdf2image</li> <li>easyocr</li> <li>pytesseract</li> <li>opencv-python</li> <li>pyzbar</li> <li>pillow</li> <li>torch</li> <li>numpy</li> </ul> <hr/> <h2>Installation</h2> <ol> <li>Clone the repository</li> <li>Create a virtual environment</li> <li>Install dependencies</li> <li>Edit a configuration file in the configs directory</li> <li>Run the engine you need</li> </ol> <h3>Example</h3> <pre> python engines/delivery_note.py </pre> <hr/> <h2>License</h2> <p> MIT License </p>
